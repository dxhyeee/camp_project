1. 문제 정의 및 데이터/모델 구성 (Step 1)
AI 과제의 첫 번째 단계로, 해결하고자 하는 문제를 정의하고 최적의 모델과 데이터를 설계합니다.
• 주제 선정: LLM(대규모 언어 모델) 기반의 "슈퍼컴퓨팅 대규모 판례 분석 및 맞춤형 추천 AI"
• 문제 정의: 일반인이 수만 건의 법률 데이터 속에서 본인에게 적합한 판례를 찾는 것은 매우 어렵습니다. 단순 키워드 검색의 한계를 극복하기 위해 문맥을 파악하는 **LLM(대규모 언어 모델)**을 도입하여 검색 효율을 극대화합니다.
• 데이터 구성: 12개의 프로세스가 균등하게 연산을 분담할 수 있도록 12,000건의 가상 판례 데이터를 .csv 형식으로 준비합니다. 해당 데이터는 리눅스 내 특정 경로(예: /home/user/data/cases.csv)에 저장하여 파이썬 프로그램이 직접 로드하도록 설계합니다.
• 모델 선택: 텍스트의 의미를 고차원 벡터로 변환하는 LLM Embedding 모델을 사용합니다. 이를 통해 문장 간의 단순 일치가 아닌, 법률적 맥락에 따른 의미적 유사성을 분석합니다.

2. 실험 수행 및 결과 분석 (Step 2)
데이터 전처리부터 AI 알고리즘 적용, Retson 장비의 12개 프로세스를 활용한 병렬 연산 과정입니다.
[AI 및 병렬 처리 메커니즘]
1. 데이터 전처리: 파이썬 pandas 라이브러리를 활용해 데이터를 로드하고, AI 분석 효율을 높이기 위한 텍스트 정제(불용어 제거 및 정규화)를 수행합니다.
2. AI 알고리즘: 사용자 입력 사건과 각 판례 벡터 간의 **코사인 유사도(Cosine Similarity)**를 계산하여 가장 유사한 판례를 도출합니다.
3. HPC 병렬화 전략 (MPI 활용):
• 데이터 분산 (MPI_Scatter): 마스터 프로세스(Rank 0)가 12,000건의 데이터를 12개 코어에 1,000건씩 균등하게 분배합니다.
• 병렬 연산: 12개의 CPU 코어가 각자 할당받은 영역에서 유사도를 동시에 계산하여 전체 연산 시간을 획기적으로 단축합니다.
• 결과 수집 (MPI_Gather): 각 프로세스가 계산한 최적의 판례 정보를 마스터 프로세스에게 보고하고, 최종적으로 가장 유사도가 높은 TOP 5 판례를 선정합니다.

3. 발표 자료 작성 및 성능 분석 (Step 3)
실험 결과를 해석하고 시각화하여 최종 발표 자료(PPT)를 구성합니다.
• 성능 비교 분석: 단일 프로세스(Serial) 실행 시와 Retson의 12개 프로세스 병렬 처리 시의 성능 차이를 심층 비교합니다.
• 결과 시각화:
• 속도 향상 그래프: 프로세스 수(1, 4, 8, 12) 증가에 따른 실행 시간 단축 및 **가속비(Speedup)**를 그래프로 제시합니다.
• AI 추천 사례 비교: 사용자의 질문과 AI가 추천한 판례가 실제 문맥적으로 얼마나 일치하는지 보여주는 비교 표를 삽입합니다.
• 결론 및 소감: LLM과 슈퍼컴퓨팅(HPC) 기술을 결합하여 실질적인 사회 문제를 해결하는 과정에서 얻은 기술적 이해와 병렬 컴퓨팅의 필요성을 기술합니다.
